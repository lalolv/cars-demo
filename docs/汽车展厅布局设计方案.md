# 汽车展厅布局设计方案（V1）

## 1. 设计目标

- 让车辆始终是视觉主角，展厅元素只做衬托和导视。
- 支持多车型切换，不需要频繁改主场景结构。
- 兼顾移动端性能，优先稳定帧率和可维护性。
- 支持后方大屏图片轮播（PPT 风格）增强品牌叙事。

## 2. 空间布局

### 2.1 总体结构（推荐）

- 中央主舞台：圆形旋转台，放置车辆展示。
- 环形视觉区：围绕舞台布置灯柱与导视物件，形成空间层次。
- 后场品牌区：设置一块主大屏作为品牌叙事核心。
- 前场观察区：为相机保留主观察角度（车头 3/4 视角）。

### 2.2 尺寸建议（可按资产比例微调）

- 舞台半径：`2.2m ~ 2.8m`
- 车与大屏距离：`10m ~ 14m`
- 主屏尺寸：`16:9`（推荐 `9.6m x 5.4m`）
- 主屏中心高度：略高于车顶（约 `2.4m ~ 3.0m`）

### 2.3 地板与墙面设计（落地版）

#### 地板（Floor）

- 目标风格：低饱和深灰镜面地台，强调车身倒影但不过度刺眼。
- 几何建议：
  - 中央圆形舞台保持现有结构（`Stage/Platform`）。
  - 外圈补一层大面积地面（建议直径 `26m ~ 32m`），作为展厅主地坪。
- 材质建议（Godot StandardMaterial3D）：
  - `albedo_color`：`#2A2D33 ~ #333840`
  - `metallic`：`0.25 ~ 0.40`
  - `roughness`：`0.18 ~ 0.32`
  - `specular`：`0.55 ~ 0.75`
  - 可选轻微法线贴图（强度 `0.08 ~ 0.15`），避免“纯平塑料感”。
- 移动端约束：
  - 避免屏幕空间反射（SSR）依赖，优先用材质高光+灯光塑形。
  - 纹理分辨率建议不超过 `2K`，且尽量复用贴图。

#### 墙面（Walls）

- 目标风格：后场“品牌主墙 + 参数副墙”，两侧弱化，形成视觉收束。
- 结构建议：
  - `BrandWall`：位于主屏后方或同层背板，宽 `12m ~ 18m`，高 `4m ~ 6m`。
  - `SpecWall`：位于后场侧后方，宽 `6m ~ 10m`，高 `3m ~ 4.5m`。
  - 两侧可加窄墙翼（宽 `2m ~ 4m`）做空间包裹，避免“悬空感”。
- 材质建议：
  - 主墙：哑光深色（`roughness 0.55 ~ 0.75`），降低反光干扰。
  - 参数墙：中性灰或品牌辅色低饱和版本，保证文字/图标可读。
  - 墙面尽量不用高频细纹理，防止移动端闪烁与锯齿感。

#### 色彩与对比控制

- 地板亮度 < 车漆高光亮度，墙面亮度 < 地板高光亮度。
- 屏幕是最亮背景元素，但峰值不超过车辆关键高光区域。
- 推荐对比顺序：`车体（主） > 屏幕（次主） > 地板高光 > 墙面`。

#### Godot 节点落地建议

- 地板：新增 `FloorOuter`（`MeshInstance3D`），使用 `CylinderMesh` 或低分段 `PlaneMesh`。
- 墙面：
  - `BackScreenRoot/BrandWall`（已接入）
  - `BackScreenRoot/SpecWall`（已接入）
  - 可选新增 `LeftWingWall`、`RightWingWall`（`MeshInstance3D`）
- 材质命名建议：`M_Floor_Base`、`M_Wall_Brand`、`M_Wall_Spec`，便于后续主题化替换。

#### 快速实施步骤（30~60 分钟）

1. 在 `Main` 下新增外圈地面 `FloorOuter`，先用纯色材质确认比例与构图。
2. 调整 `FloorOuter` 的 `metallic/roughness/specular` 到推荐区间，确保有倒影但不抢车。
3. 调整 `BrandWall/SpecWall` 尺寸与位置，使其在主镜头中不遮挡车轮廓。
4. 锁定主镜头后微调墙面明度（先降亮度再提对比），保证文案可读。
5. 在移动端预览检查闪烁、锯齿和帧率，必要时降低贴图与灯光复杂度。

## 3. 灯光设计

采用三层灯光体系：基础光、造型光、氛围光。

### 3.1 基础光（空间可见）

- `WorldEnvironment` 低强度环境光，避免暗部死黑。
- 保留一盏低强度主方向光，统一空间方向感。

### 3.2 造型光（强调车身）

- 前上方左右各一盏 Key SpotLight，突出引擎盖和车头线条。
- 后侧一盏 Rim SpotLight，拉出车身轮廓和尾部层次。
- 移动端仅关键灯开启阴影，其他灯尽量关闭阴影。

### 3.3 氛围光（空间风格）

- 后场大屏与灯柱负责氛围，不与车辆争主视觉。
- 主色建议 1 个，辅色建议 1 个，避免高饱和多色混杂。

## 4. 大屏轮播设计（图片 PPT 方案）

### 4.1 播放策略

- 内容类型：品牌主视觉、车型卖点页、功能亮点页。
- 轮播节奏：每页 `5s ~ 8s`，切页淡入 `0.4s ~ 0.8s`。
- 循环模式：默认循环，支持随机起始页。

### 4.2 视觉控制

- 大屏亮度低于车身高光，防止喧宾夺主。
- 页面排版以大图+短文案为主，减少高频闪烁动效。

### 4.3 性能建议（移动端）

- 统一图片分辨率（如 `1920x1080`），避免混合规格。
- 优先图片轮播，不上实时视频解码。
- 可在非主视角时降低更新频率（后续优化项）。

## 6. 分阶段实施计划

### 阶段 A（已实现）

- 新增后场主大屏节点（MainScreen）。
- 新增图片轮播脚本（ScreenSlideshow）。
- 新增三盏展车造型灯（KeyLeft/KeyRight/Rim）。

### 阶段 B（进行中）

- 加入品牌墙与参数墙几何体。
- 调整地面材质反射与粗糙度，提升高级感。
- 补充 UI 开关（大屏开关、轮播速度）。

当前进度：以上 3 项已完成第一版接入，后续进入视觉微调与性能校准。

### 阶段 C（优化）

- 增加主题配置（科技蓝/经典暖金等）。
- 增加镜头预设位（前 3/4、侧面、车尾）。
- 针对移动端做画质分级和灯光降级。

## 7. 内容素材清单（建议）

- 必选：品牌封面、车型总览、核心卖点、收尾页。
- 可选：技术参数页、内饰细节页、智能功能页。
- 命名规范：`slide_01_cover.png`、`slide_02_design.png`。
